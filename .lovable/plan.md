

## Problem

If an admin clicks "Activate" twice quickly, or clicks it again after the first call completes (ACH takes days to settle, so the button may reappear), the edge function could attempt to pay the same invoice twice — or pay a second invoice generated by Stripe.

## Plan

### 1. Server-side idempotency guard in `activate-subscription/index.ts`

After retrieving the Stripe subscription status (line 101), add a check: if the subscription status is already `active`, return a success response immediately without attempting to pay again. This covers the case where a second call arrives after Stripe has already processed the first payment.

For the case where the invoice is still processing (ACH pending), `stripe.invoices.pay()` on an already-processing invoice will throw an error — but we should handle it gracefully. After the `stripe.invoices.pay()` call, catch the specific Stripe error for "invoice is not open" and return a friendly message instead of a generic error.

Additionally, before calling `stripe.invoices.pay()`, re-check the invoice status is still `open`. If it's `paid` or `processing`, return early with a "already activated" message.

### 2. Client-side: hide the Activate button after successful activation

In `src/pages/admin/Billing.tsx`, after `handleActivateSubscription` succeeds, the query invalidation already refreshes the data. But the `isReadyToActivate` logic may still show the button before billing history updates. Add the successfully-activated subscription ID to a local `Set` state so the button is hidden immediately after the first click succeeds.

### Files to change

- **`supabase/functions/activate-subscription/index.ts`** — Add guard for already-active subscriptions and already-paid/processing invoices
- **`src/pages/admin/Billing.tsx`** — Track activated subscription IDs in local state to immediately hide the button after success

### Technical details

Edge function changes (around line 101):
```typescript
// If subscription is already active, return success without double-charging
if (stripeSubscription.status === "active") {
  return Response(JSON.stringify({
    success: true,
    message: "Subscription is already active",
    alreadyActive: true
  }), ...);
}
```

Before `stripe.invoices.pay()` (around line 158), re-fetch invoice status:
```typescript
const freshInvoice = await stripe.invoices.retrieve(invoiceId);
if (freshInvoice.status !== "open") {
  return Response(JSON.stringify({
    success: true,
    message: "Payment already initiated",
    alreadyActive: true
  }), ...);
}
```

Client-side in Billing.tsx, add state:
```typescript
const [activatedIds, setActivatedIds] = useState<Set<string>>(new Set());
```
After successful activation, add the ID to the set. In the `isReadyToActivate` check, exclude IDs in `activatedIds`.
